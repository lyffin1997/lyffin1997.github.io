Snmp_exporter安装部署

1. 开机交换机snmp协议（非华为交换机跳过这一步）

​

//192.168.1.9是snmp_exporter地址

2. 配置go 环境

安装go1.15.6（tar包或者命令行安装都行）

vim /etc/profile

export GOROOT=/usr/local/go

export PATH=$PATH:$GOROOT/bin

export GOPATH=/root/go

source /etc/profile

go version #显示版本号即安装成功

//go环境变量国内代理：

go env -w GO111MODULE=on

go env -w GOPROXY=https://goproxy.cn,direct

3. 下载snmp环境

yum install gcc gcc-g++ make net-snmp net-snmp-utils net-snmp-libs net-snmp-devel

(此外，git也需安装配置好)

//测试snmp

snmpwalk -v 2c -c public 192.168.1.222 .1

//pubilc是团体名，.1表示查看所有信息，192.168.1.222是设备ip

4. 下载generator代码

go get github.com/prometheus/snmp_exporter/generator

5. 编译

//路径不一定准确，总之找到GOPATH路径下的generator文件夹

cd ${GOPATH-$HOME/go}/src/github.com/prometheus/snmp_exporter/gener
ator

go build

6. 使用generator生成snmp.yml文件

//将所需监控交换机厂商提供的mib包放入到${GOPATH-$HOME/go}/src/github.com/prometheus/snmp_exporter/generator/mibs目录中

//输入以下命令查看oid名字，最后为交换机厂家提供的mib库名称

snmptranslate -Tz -m ./mibs/TRPS-9168GT-M12X-BP2-24V_v100.mib

//选中对应的oid进行测试，如：

snmpwalk -v 2c -c public 192.168.1.222 1.3.6.1

//若无报错，然后要修改generator.yml文件，oid或名称都可以，写法不一样

vim generator.yml

写法1:

modules:
  # Default IF-MIB interfaces table with ifIndex.
  if_mib:
    walk:
      - 1.3.6.1
    version: 2
    timeout: 30s
    auth:
      community: public

写法2:

modules:
  # Default IF-MIB interfaces table with ifIndex.
  if_mib:
    walk: [internet]
    version: 2
    timeout: 30s
    auth:
      community: public

//If_mib:模块名，可自定义

//Walk:所需监控指标的oid（得自己查），如上图所示

//version: snmpv2版本

//Community:团体名（与第一步中的团体名一致）

//其他配置保持原样就行

//指定mib库，注意在generator文件夹下执行该命令

export MIBDIRS=mibs

//生成snmp.yml文件

./generator generate

7. 运行snmp_exporter

//将1.6中生成的snmp.yml文件复制并替换下载的snmp_exporter中的snmp.yml

//snmp_exporter下载地址：https://github.com/prometheus/snmp_exporter

//将生成的snmp配置移到snmp_exporter目录下

cp snmp.yml /usr/local/snmp_exporter/

//指定配置文件运行

./snmp_exporter --config.file=/usr/local/snmp_exporter/snmp.yml

8. 防火墙添加对外暴露端口（在部署exporter的机器上）

firewall-cmd --zone=public --permanent --add-port=9116/tcp

firewall-cmd --reload 

firewall-cmd --list-port

9. 验证

访问192.168.1.9:9116

//192.168.1.9是部署snmp_exporter的ip地址

//9116是snmp_exporter默认端口号

11. 修改prometheus.yml文件（注意，实际配置文件要把注释都删掉，否则prometheus启动会报错）

- job_name: 'snmp_exporter'
    # 采集超时：10s
    scrape_interval: 10s
    static_configs:
      - targets:
        - 192.168.1.222  #交换机ip
    metrics_path: /snmp
    params:
      module: [huawei_mib]  #修改成2.4步自己设置的mib名称
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - source_labels: [__param_target]
        target_label: instance
      - target_label: __address__
        replacement: 192.168.0.254:9116  # 安装snmp_exporter主机的ip和端口号

//192.168.1.222是交换机ip地址

//[huawei_mib]：是1之前generato中填的module名

12. 重启prometheus并验证
