2. springboot是什么

2.1 spring目的、理念

Spring是为了解决企业级应用开发的复杂性而创建的，简化开发

2.2 spring如何简化开发

  • 基于POJO的轻量级和最小侵入性编程（bean）
  • 通过IOC，依赖注入（DI）和面向接口实现松耦合（核心：容器）
  • 基于切面（AOP）和惯例进行声明式编程（核心：AOP）
  • 通过切面和模板减少样式代码

2.3 springboot是什么

  • springboot就是一个javaweb的开发框架，和springmvc类似。

  • 正式定义：简化开发，约定大约配置（核心思想，在maven、spring、springmvc、springboot、docker、k8s……中都遵循该思想） ，能迅速的开发web应用，几行代码开发一个http接口。

  • 所有的技术框架都遵循一条主线规律：从一个复杂的应用场景，衍生一种规范框架

  • java企业级应用发展：J2EE -> spring -> springboot

  • 默认帮和我们进行很多设置，只需要我门进行很少量的spring配置，同时集成了大量常用的第三方库配置（Redis、MongoDB、Jpa、RabbitMQ、Quartz等）

2.4 springboot优点

  • 为所有spring开发者更快的入门
  • 开箱即用 ，提供各种默认的配置来简化项目配置
  • 内嵌式容器简化Web项目
  • 没有冗余代码生成和XML配置的要求

2.5 什么是微服务

微服务是一种架构风格（MVC、MVVM都是一种架构）。要求在开发应用时，将应用构建成一系列小服务的组合，可以通过http（以及rpc等）的方式进行沟通。

​

2.5.1 单体架构

单体架构（all in one ）是指，我们将一个应用中的所有应用服务都封装在一个应用中。（把数据库访问、web访问等各个功能放到一个war包中，如ssm框架）

优点：

  • 易于开发和测试，也方便部署。扩展时，只需将war包复制到各个服务器上，再做负载均衡即可。

缺点：

  • 维护难：哪怕只修改一行代码，都需要停掉整个服务，重新部署war包。

2.5.2 微服务架构

打破all in one架构方式，把每个功能元素独立出来，把每个功能元素独立出来，然后将独立出来的功能元素根据需求动态组合。

​

https://martinfowler.com/articles/microservices.html

https://www.cnblogs.com/liuning8023/p/4493156.html (翻译版)

微服务架构是对功能元素进行复制，而没有对整个应用复制。

优点：

  • 节省了调用资源
  • 每个功能元素的服务都是一个可替换、可独立升级等软件代码（高内聚，低耦合）

缺点：

  • 运维难度增大，原来只需维护tomcat和war包，现在需要维护所有服务、以及负载均衡等一系列问题（但是spring推出了解决方案，后续会提到）

2.5.3 如何构建微服务

大型微服务架构中，每个功能元素（即微服务）各自完成自己的功能，然后通过http请求相互调用（如咱们的IOT2、大运项目）。若需升级修改，只需更新升级其中一个功能服务单元，而不需要停掉整个应用。

  • spring推出的大型分布式微服务等全套、全程产品

  • springboot（build anything）可以快速的帮我们构建一个个功能独立的微服务应用单元

  • springcloud（coordinate anything）帮我们实现大型分布式网络服务的调用

  • spring cloud data flow（connect everything）帮我们在分布式中，进行流式数据计算、批处理

  • spring为我们想清楚了整个从构建应用到大型分布式应用的全流程方案

用@Controller标识的类代表控制器类（控制层），可以调用用@Service标识的类

控制器由spring管理，在applicationContext.xml中如下

 <context:component-scan base-package="com.byd...">
   </context:component-scan>

扫描器会扫描该包下所有@Component、@Controller等注解，并将他们添加到IOC容器中，由IOC容器来管理。

  • @RestController = @Controller+@ResponseBody

  • @ResponseBody 规定返回值为json字符串，用于标识方法，注意是返回数据不是返回页面

  • @RequestMapping给出外界访问方法路径，用于出发请求

  • parent:继承spring-boot-starter-parent的依赖管理，控制版本与打包等内容

  • dependencies:项目具体依赖

  • starter:启动器

  • spring-boot-starter-web 实现http接口（包含了springmvc）
