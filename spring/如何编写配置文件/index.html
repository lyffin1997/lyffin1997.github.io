如何编写配置文件

1. 语法

  • yml和yaml一样的文件格式，但是yaml优先级比yml高

  • 名称固定：application.properties或者application.yml

  • 可以同时存在properties和yml，且都会生效

  • 语法:

    • application.properties: key=value
server.port=8080
    • application.yml: key:空格value
server:
    port: 8080

  • yml可以存对象，properties不行

    • yml
student:
    name: lifeng
    age: 18 
#行内写法
student: {name: lifeng,age: 18}

    properties

student.name=lifeng
student.age=18
  • yml可以写数组
pets:
    - cat
    - dog
    - pig
    
pets: [cat,dog,pig]
  • yml可以用特殊表达式（可搜索spring el表达式）
name: ${random.uuid}
age: ${random.int}


person:
    h: a


eee: ${person.h}
//若不存在，则需要自己写,如下面写法，bbb的值应当为mmm_ddd，若m存在，则优先取存在的值
bbb:${person.m:mmm}_ddd

2. YAML与xml

  • yml配置：
server:
   port: 8080
  • xml配置
<sever>
    <port>8080<port>
</sever>

3. 给实体类赋值

传统方法需要new实例来赋值，spring简单点可以用@value注解来赋值，但是需要每个字段都用@value，springboot更简单，只需要用@ConfigurationProperties标识类即可（可能会报红，但不影响代码运行，点击错误提示进入网站后它会告诉你需要添加一个依赖，添加到pom文件中即可）

  • @ConfigurationProperties

将配置文件中配置到每一个属性到值映射到这个组件中（即用注解标识到类）；告诉springboot将本类中所有属性和配置文件中匹配的配置进行绑定，绑定方法：prefix = “xxx”

前提条件：该组件是容器中的组件（即用了@component注解到类）

  • Person.java
@Component
@ConfigurationProperties(prefix = "person")
public class Person {
    private String name;
    private Integer age;
    
    //必须要创建好无参构造和有参构造，否则可能会报错
    public Person() {
    }
    public Person(String name, Integer age) {
        this.name = name;
        this.age = age;
    }
    //然后还要创建号get、set、toString方法
    ……
    }
  • yml
person:
    name: lifeng
    age: 18
  • 功能延伸：给实体类赋值的功能，可以把以前需要写到配置文件中的配置信息绑定到java文件中（至于为什么要放到java文件中，看项目具体需求，有的需要这样做）

4. 指定配置文件赋值

  • @PropertySource，但是得用@Value一个个配置
  • Person.java
@Component
@PropertySource(value = "classpath:lifeng.properties")
public class Person {
    //SPEL表达式取出配置文件值
    @Value("${name}")
    private String name;
    private Integer age;
    
    //必须要创建好无参构造和有参构造，否则可能会报错
    public Person() {
    }
    public Person(String name, Integer age) {
        this.name = name;
        this.age = age;
    }
    //然后还要创建号get、set、toString方法
    ……
    }
  • lifeng,properties

值是中文的话把这个勾上，否则会是乱码

name=黎峰

5. @ConfigurationProperties和@Value

  • 松散绑定：yml中last-name等价于lastName，-后面的字母默认大写

  • JSR303数据校验：在字段中增加一层过滤器验证，保证数据合法性

    • @Validated：用于标识类，在创建实例时会自动校验字段中标识了相应注解的字段
@Validated
public class Person {
    @Email(message="name格式错误") //校验字段是否为邮箱格式
    private String email;
    //@Null、@NotNull、@Length等等
}
  • 复杂型封装：可以封装对象

6. 多环境配置

  • 在项目根目录config文件夹下或项目根目录下，可以写application.yml文件
  • 类路径（resources）下到config文件夹或者类路径下可以写配置文件(classpath指resources目录下)
  • 优先级：项目路径>类路径；config>同级目录
